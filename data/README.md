# Data provenance

Everything under `data/` is either downloaded real public data (`data/raw/`,
gitignored, fetched by the pipeline) or derived from it (`data/processed/`,
gitignored, regenerated by `make data`). Nothing here is synthetic. Config
values referenced below live in `experiments/configs/data.yaml`.

## Road topology — OpenStreetMap

Downloaded automatically by `roar/graph/load_graph.py` via OSMnx, for the
bounding box in `experiments/configs/data.yaml` (`osm.bbox`) — the extent of
the METR-LA sensor network plus a small buffer. Cached to
`data/processed/la_drive_graph.graphml` and `data/processed/edges.parquet`.

No manual step needed as long as the machine running `make data` has internet
access to the Overpass API. If Overpass is unreachable, OSMnx will raise —
rerun once connectivity is restored; nothing is substituted automatically.

## Traffic speeds — METR-LA

METR-LA is the loop-detector speed dataset from Li et al., *"Diffusion
Convolutional Recurrent Neural Network,"* ICLR 2018: 207 sensors on LA
highways, 5-minute average speeds (mph), 2012-03-01 00:00 through
2012-06-27 23:55 (119 days, 34,272 time buckets, verified against the
downloaded file — see `roar/graph/features.py`).

**Canonical source:** the original authors distribute the file
(`metr-la.h5`, pandas HDF5) via the links in
[liyaguang/DCRNN](https://github.com/liyaguang/DCRNN):

- Google Drive: <https://drive.google.com/open?id=10FOTa6HXPqX8Pf5WRoRwcFnW9BrNZEIX>
- Baidu Yun: <https://pan.baidu.com/s/14Yy9isAIZYdU__OYEQGa_g>

Both require manual, browser-based download (Google Drive/Baidu do not serve
large files to unauthenticated scripted requests), which would break
`make data`'s "one command, no manual steps" requirement.

**What the pipeline actually downloads:** a byte-identical-in-content CSV
mirror of the same speed matrix, published at
[hazdzz/dcrnn_data](https://github.com/hazdzz/dcrnn_data) (LGPL-2.1),
directly fetchable over plain HTTPS:

- `https://raw.githubusercontent.com/hazdzz/dcrnn_data/main/metr_la/vel_metr_la.csv`
  (207 sensor columns + timestamp index, mph, verified against the known
  METR-LA shape: 34,272 rows x 207 sensors, 2012-03-01 to 2012-06-27).

Sensor metadata (real lat/lon per sensor, and real road-network distances
between sensor pairs, both computed by the original DCRNN authors from the
actual LA road network) comes directly from the `liyaguang/DCRNN` repo
(MIT-licensed), also plain HTTPS, no auth:

- `https://raw.githubusercontent.com/liyaguang/DCRNN/master/data/sensor_graph/graph_sensor_locations.csv`
- `https://raw.githubusercontent.com/liyaguang/DCRNN/master/data/sensor_graph/distances_la_2012.csv`

All three URLs are pinned in `experiments/configs/data.yaml` under
`metr_la:`. `roar/graph/map_sensors.py` downloads and caches them into
`data/raw/` on first run.

**If any of these URLs go dead:** `make data` will fail loudly (network
error), not substitute synthetic speeds. In that case, download `metr-la.h5`
manually from the Google Drive/Baidu links above, convert it to the same CSV
layout with `pandas.read_hdf(...).to_csv(...)`, and place the result at
`data/raw/vel_metr_la.csv` before rerunning `make data`.
